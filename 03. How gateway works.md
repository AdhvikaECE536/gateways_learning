EXAMPLE: Laptop sends packet to google.com

### STEP1:
PACKET ORIGINATES

Laptop creates an IP source. 

-  Source: 192.168.1.50. Destination: 142.250.77.78 (Google's IP)
-  It first checks its own routing table- is the destination in local subnet (192.168.1.x)? no.

### STEP2: 
ARP for the gateway

- Laptop knows that the default gateway IP is so and so, but needs its MAC address to send the frame
- Broadcasts an ARP request: "whos 142.250.77.78"; the gateway replies with its MAC. This happens only once; the MAC is catched

### STEP3:
Packet arrives at gateway

- The ethernet frame arrives
- Gateway strips the L2 header and reads the L3 IP packet. It checks its routing table for the best route to 142.250.77.78

### STEP4:
NAT translation 

- Your laptop has a private IP (RFC 1918- not routable on the internet).
- The gateway rewrites the source IP from 192.168.1.50 to public IP and maps this in a NAP table with port numbers. This is called NAPT/PAT

### STEP5:
Forwarded upstream

- Gateway forwards the modified packet to ISP router, which routes it hop by hop toward Google's servers
- Each router along the way only cares about the destination IP

### STEP6:
Response returns

- Google's response arrives at gateway addressed to public IP
- Gateway looks up the NAT table, translates the destination back to 192.168.1.50 port and forwards the frame back to the laptop

### STEP7:
Laptop receives

- Your laptop receives the packet. All the NAT translation was invisible- this is the entire point
